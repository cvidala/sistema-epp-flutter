    **2. Create `test/integration/supabase/helpers/test_client.dart`**

    This file provides the per-role client factory. All tests import this and use these
    helpers instead of constructing clients directly.

    Key design constraints (from research, per D-01 approach decision):
    - NEVER use `Supabase.instance.client` — it carries shared session state between tests.
    - Use `SupabaseClient(url, anonKey)` directly for isolated instances.
    - Service role key comes from `Platform.environment['SUPABASE_SERVICE_ROLE_KEY']`
      — throw a descriptive `StateError` if missing.
    - Anon client: `SupabaseClient(url, anonKey)` with NO `signInWithPassword` call.

    Exports (all top-level functions, not a class — matches project convention of direct
    function exports for test utilities):
    - `const String kTestPrefix = 'test_qa_'` — sentinel prefix for all test data
    - `Future<SupabaseClient> clientForRole(String role)` — role values: 'admin',
      'supervisor', 'readonly'. Looks up credentials from a private const map
      (emails/passwords match Task 0 setup). Creates a fresh `SupabaseClient` and calls
      `signInWithPassword`. Returns the authenticated client.
    - `SupabaseClient serviceClient()` — returns `SupabaseClient(url, serviceRoleKey)`
      where `serviceRoleKey = Platform.environment['SUPABASE_SERVICE_ROLE_KEY']!`.
    - `SupabaseClient anonClient()` — returns `SupabaseClient(url, anonKey)` with no
      sign-in. For kiosk simulation (RLS-04).
    - `Future<void> disposeClient(SupabaseClient client)` — calls
      `client.auth.signOut()` then `client.dispose()` for tearDown.

    Import `dart:io` for `Platform.environment`. Import `package:supabase_flutter/supabase_flutter.dart`.
    Use `SupabaseConfig.url` and `SupabaseConfig.anonKey` from `'../../../lib/config/supabase_config.dart'`
    (relative path from `test/integration/supabase/helpers/`).

    - `static Future<void> tearDownTestData()` — deletes all rows where

      `local_event_id LIKE 'test_qa_%'` from `stock_movimientos` if the column exists.

      `audit_log` rows where `registro_id` IN (ids cleaned above) — optional, reduces
